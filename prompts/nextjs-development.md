## Next.js 개발 공통 프롬프트

- 기본 원칙: 안정성/가독성 우선, 불필요한 클라이언트 상태/렌더링 최소화.
- 구조: App Router 기준, 라우트 그룹과 레이아웃 분리를 명확히 한다.
- 컴포넌트: Server Component 우선, 필요할 때만 `use client` 사용.
- 데이터 패칭: 서버 액션/서버 패칭 우선, 캐시는 명시적으로 설정한다.
- 상태 관리: 로컬 상태 우선, 전역 상태는 필요 시에만 도입한다.
- 성능: `next/image`/`next/font` 사용, 번들 크기/클라이언트 JS 최소화.
- 보안: 환경변수 노출 금지, 입력값 검증 및 인가 체크를 경계에서 수행.
- 패키지 매니저: 기본은 Yarn 사용 (`yarn`, `yarn lint`, `yarn build`, `yarn test`).
- 테스트: 변경이 행동/버그에 영향 있으면 최소 단위/통합 테스트 추가; 없으면 이유를 명시.
- 품질 체크: 가능한 경우 `yarn lint`, `yarn tsc`, `yarn test` 실행 제안.
- 문서화: 공개 API/중요 UI/UX 변경 시 README 또는 코드 주석 업데이트.
