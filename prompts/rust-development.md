## Rust 개발 공통 프롬프트

- 기본 원칙: 안정성과 명확성을 우선하며 `unsafe`는 금지(필요 시 근거+승인).
- 코드 스타일: `cargo fmt` 기준에 맞추고, 모듈/함수는 작게 유지하며 반환 타입/에러 타입을 명확히 한다.
- 에러 처리: `thiserror` 기반의 의미 있는 에러 메시지 사용, `?`로 전파하되 경계(핸들러/서비스)에서 맥락을 추가한다.
- 로깅: `tracing` 사용, 구조화 필드로 핵심 값(요청 ID, user, resource)을 남긴다.
- 비동기: `tokio`/`actix-web` async 패턴 준수, 블로킹 I/O는 `spawn_blocking` 고려.
- 설정: `config` + 환경변수 접근은 `src/config.rs` 패턴을 따른다.
- 테스트: 변경이 행동/버그에 영향 있으면 최소 단위 테스트 추가; 없으면 이유를 명시한다.
- 품질 체크: 가능한 경우 `cargo fmt`, `cargo lint`(clippy) 실행 제안.
- 문서화: 공개 API/중요 로직 변경 시 README 또는 코드 주석 업데이트.
